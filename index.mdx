takeovflow is a command-line tool built for bug bounty hunters and security researchers that automates subdomain takeover detection. It combines passive subdomain discovery, active HTTP analysis, and concurrent CNAME pattern matching to surface vulnerable subdomains pointing at unclaimed cloud services — and generates structured Markdown and JSON reports for triage.

## How takeovflow works

takeovflow runs three phases in sequence, each building on the previous:

<Steps>
  <Step title="Passive discovery">
    Enumerate subdomains using `subfinder` and `assetfinder`. Results are deduplicated and written to a temporary file for the next phase.
  </Step>
  <Step title="Active scanning">
    Resolve discovered subdomains with `dnsx`, probe HTTP services with `httpx`, then confirm takeover candidates with `subjack` and `nuclei`. All phases run concurrently.
  </Step>
  <Step title="CNAME fingerprinting">
    Each subdomain is checked against 55 cloud provider CNAME patterns using concurrent `dig` calls. Matches are assigned severity levels and deduplicated across all detection sources.
  </Step>
  <Step title="Report generation">
    Results are written to a timestamped Markdown report and optionally a JSON file, filtered by your chosen minimum severity level.
  </Step>
</Steps>
