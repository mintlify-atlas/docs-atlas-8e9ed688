This guide walks you through running your first subdomain takeover scan with takeovflow. By the end you will have discovered subdomains for a target domain, checked them for CNAME takeover vulnerabilities, and produced a Markdown report you can use for triage.

## Prerequisites

Before you start, make sure you have:

- Python 3.8 or later installed
- At least one of `subfinder` or `assetfinder` installed (required for passive discovery)
- `dig` installed (required for CNAME fingerprinting)

See the [installation guide](/installation) for the full list of tools and how to install them.

## Steps

<Steps>
  <Step title="Clone and set up takeovflow">
    If you have not already cloned the repository, do that now and make the script executable.

    ```bash
    git clone https://github.com/theoffsecgirl/takeovflow.git && cd takeovflow && chmod +x takeovflow.py
    ```

    For detailed dependency installation instructions, see [Install takeovflow and its dependencies](/installation).
  </Step>

  <Step title="Run a basic scan">
    Run a full passive + active + CNAME scan against a single domain. The `-v` flag enables verbose output so you can follow the scan as it progresses.

    ```bash
    python3 takeovflow.py -d example.com -v
    ```

    takeovflow will print the banner, check which tools are available, then start passive discovery followed by active scanning and CNAME fingerprinting.
  </Step>

  <Step title="Read the terminal output">
    As the scan runs, takeovflow prints findings in real time. A typical output looks like this:

    ```text
    [*] Domains loaded: 1
    [*] Running passive discovery: subfinder, assetfinder
    [*] Subdomains found: 1247
    [*] Resolving CNAMEs (concurrent)...

    [!] POTENTIAL TAKEOVER → blog.example.com
        CNAME : example.github.io
        Service: GitHub Pages
        Severity: HIGH

    [!] POTENTIAL TAKEOVER → cdn.example.com
        CNAME : example.s3.amazonaws.com
        Service: AWS S3 / Elastic Beanstalk
        Severity: HIGH

    [~] INVESTIGATE → api.example.com
        CNAME : example.herokudns.com
        Service: Heroku
        Severity: HIGH

    [+] CNAME findings: 3
    [+] Report saved → takeovflow_report_20240416_1523.md
    [+] JSON saved   → takeovflow_report_20240416_1523.json
    ```

    `[!] POTENTIAL TAKEOVER` entries are HIGH or CRITICAL severity findings that should be manually verified. `[~] INVESTIGATE` entries are candidates that need further review before claiming.
  </Step>

  <Step title="Find your report">
    When the scan finishes, takeovflow writes a Markdown report to the current working directory. The filename follows the format:

    ```text
    takeovflow_report_YYYYMMDD_HHMM.md
    ```

    For example, a scan started at 15:23 on 16 April 2024 produces `takeovflow_report_20240416_1523.md`. The report contains a summary table with totals for subdomains discovered, DNS-resolved hosts, active HTTP services, and potential takeovers — followed by per-domain finding tables sorted by severity.

    To save reports to a specific directory instead of the current working directory, add `--output-dir ./reports` to your command.
  </Step>

  <Step title="Generate JSON output">
    To also produce a machine-readable JSON report alongside the Markdown file, add the `--json-output` flag.

    ```bash
    python3 takeovflow.py -d example.com -v --json-output
    ```

    The JSON report uses the same filename format: `takeovflow_report_YYYYMMDD_HHMM.json`. It includes tool name, version, UTC start and finish timestamps, and the full structured finding data — suitable for parsing in CI pipelines or downstream tooling.
  </Step>
</Steps>

## Common scan variations

<CodeGroup>

```bash Single domain
python3 takeovflow.py -d example.com -v
```

```bash Multiple domains from file
python3 takeovflow.py -f scope.txt
```

```bash Passive discovery only
python3 takeovflow.py -d example.com --passive-only
```

```bash Active phase with known subdomains
python3 takeovflow.py --active-only --subs-file subdomains.txt -d example.com
```

</CodeGroup>
